## 1. Objetivo final (medible)

Convertir tAIger+ en un coach con memoria longitudinal determinística que (a) detecta patrones sobre el 100% de rondas, (b) elige UN patrón a trabajar con jerarquía explícita, (c) asigna un **plan estructurado y machine-readable**, (d) mide adherence y outcome ronda a ronda, (e) cierra el plan cuando se alcanza el target o el patrón se resuelve, y (f) reporta el ciclo de vida completo en un Admin Brain navegable por Juanjo.

**Meta numérica al final del sprint:**
- 100% de usuarios con ≥5 rondas tienen al menos un evento `pattern_detected` en `coach_events`.
- ≥50% tienen plan activo (`coach_plans.status='active'`).
- 100% de planes activos generan `plan_outcomes` automáticamente al entrar una ronda nueva.
- 0 alucinaciones en el log de validador post-respuesta sobre un set de regresión de 20 mensajes.
- Cache hit ratio >80% en sesiones con >3 mensajes.
- Costo <$1 USD/usuario activo/mes.

## 3. Definiciones operativas (números reales — fuente de verdad)

> Esta tabla es invariante. Si un agente la cambia, debe actualizar este doc primero.

| Término | Definición operativa |
|---|---|
| **Error en un hoyo** | `score - par >= +1` (bogey o peor). |
| **Post-error** | bogey en hoyo `i` seguido de score `>= par+1` en hoyo `i+1`. |
| **Tilt post-error / post-bogey spiral** | % de bogeys seguidos de bogey-o-peor sobre rondas con ≥10 bogeys totales. Detección ≥40%. Confianza alta ≥55%. (Ya en `patterns.ts:201-232`.) |
| **Caída en cierre / back nine collapse** | `back9_avg - front9_avg > 2.5` strokes promedio sobre últimas N rondas. (Ya en `patterns.ts:48-77`.) |
| **Alta dispersión / driving inconsistency** | Coeficiente de variación de `total_gross` sobre últimas 10 rondas `> 0.06` (≈ 5-6 strokes de desviación en un golfista de 90). **A implementar en `patterns.ts` — hoy solo está nombrado en system prompt.** |
| **Pressure deterioration** | Score promedio en últimos 4 hoyos `>` score promedio en primeros 14 + 1.5 strokes/hoyo, sobre rondas de 18. **A implementar.** |
| **Mínimo de rondas para diagnóstico** | **5 rondas finalizadas con scores hoyo a hoyo.** (Alineado con `if (rounds.length < 5) return` actual.) El gate de UI baja a 1 ronda solo para conversación, no para asignar plan. |
| **Diagnóstico creíble** | (a) ≥5 rondas finalizadas, (b) al menos 1 patrón con `confidence >= 0.6`, (c) `data_points` suficientes para la métrica. |
| **Umbrales de confidence para mostrar al usuario** | <0.5 = ocultar; 0.5-0.7 = "tendencia"; >0.7 = "patrón confirmado". |
| **Patrón resuelto** | `confidence < 0.4` durante 3 recálculos consecutivos (~3 rondas nuevas) → `player_patterns.status = 'resolved'`. Si vuelve a subir, `status = 'active'` otra vez. |
| **Adherence "full"** | `metric_value` mejoró ≥50% del gap `baseline → target`. |
| **Adherence "partial"** | `metric_value` mejoró pero <50% del gap. |
| **Adherence "none"** | `metric_value` no mejoró o empeoró. |
| **Plan resuelve por target** | `plan_outcomes` con `target_reached=true` en 3 rondas consecutivas → `coach_plans.status='resolved'`, `resolution_reason='target_reached'`. |
| **Plan expira** | `created_at + duration_days < now()` sin resolver → `status='expired'`. |
| **Plan superseded** | Decision Engine detecta otro patrón con `score > 2× score actual` durante recálculo → marca el activo como `superseded`, abre nuevo `active`. |
| **Decision Engine — prioridad** | `score = severity_weight × confidence`. severity_weight = `{critical: 3, warning: 2, info: 1}`. Desempate 1: mayor `data_points`. Desempate 2: pattern row `created_at` más antiguo. |

{
  id: 'pressure_deterioration',
  name: 'Deterioro bajo presión (cierre)',
  description: 'Score promedio en últimos 4 hoyos > resto + 1.5 strokes/hoyo, sobre rondas de 18',
  requires18Holes: true,
  severity: 'warning',
  detect: (rounds) => {
    const eligible = rounds.filter(r => Array.isArray(r.scores) && r.scores.filter(s => s != null).length === 18)
    if (eligible.length < 5) return { detected: false, confidence: 0 }
    let triggers = 0
    for (const r of eligible) {
      const scores = r.scores as number[]
      const last4 = scores.slice(14, 18)
      const rest = scores.slice(0, 14)
      const avgLast = last4.reduce((a, b) => a + b, 0) / 4
      const avgRest = rest.reduce((a, b) => a + b, 0) / 14
      if (avgLast - avgRest > 1.5) triggers++
    }
    const ratio = triggers / eligible.length
    if (ratio < 0.4) return { detected: false, confidence: ratio }
    return {
      detected: true,
      confidence: Math.min(0.95, 0.5 + ratio),
      metadata: { triggers, eligible_rounds: eligible.length, ratio },
    }
  },
  recommendation: 'Trabajar rutina pre-shot extendida en los últimos 4 hoyos. Respiración cuadrada antes del 15.',

### 5.3 — Decision Engine

**File nuevo:** `src/golf/coach/decision-engine.ts`

```ts
import type { SupabaseClient } from '@supabase/supabase-js'

const SEVERITY_WEIGHT = { critical: 3, warning: 2, info: 1 } as const

export interface PatternRow {
  id: string
  pattern_id: string
  confidence: number
  data_points: number
  severity: 'critical' | 'warning' | 'info'
  status: string
  created_at: string
}

export interface DecisionInput {
  patterns: PatternRow[]   // de player_patterns WHERE status='active'
  activePlan: { id: string; pattern_id: string; created_at: string; duration_days: number } | null
}

export interface DecisionOutput {
  winningPattern: PatternRow | null
  shouldSupersede: boolean
  reason: string
}

export function decide(input: DecisionInput): DecisionOutput {
  const { patterns, activePlan } = input
  if (patterns.length === 0) return { winningPattern: null, shouldSupersede: false, reason: 'no_active_patterns' }

  const scored = patterns.map(p => ({
    p,
    score: SEVERITY_WEIGHT[p.severity] * p.confidence,
  })).sort((a, b) => {
    if (b.score !== a.score) return b.score - a.score
    if (b.p.data_points !== a.p.data_points) return b.p.data_points - a.p.data_points
    return new Date(a.p.created_at).getTime() - new Date(b.p.created_at).getTime()
  })

  const winner = scored[0].p

  // Si hay plan activo y el winner es OTRO patrón, decidir si supersede
  if (activePlan) {
    const currentScored = scored.find(s => s.p.pattern_id === activePlan.pattern_id)
    if (!currentScored) {
      // El patrón del plan actual ya no está activo → supersede
      return { winningPattern: winner, shouldSupersede: true, reason: 'current_pattern_resolved' }
    }
    if (winner.pattern_id !== activePlan.pattern_id) {
      const winnerScore = scored[0].score
      if (winnerScore > 2 * currentScored.score) {
        return { winningPattern: winner, shouldSupersede: true, reason: 'higher_priority_pattern' }
      }
    }
    // El plan sigue siendo el correcto, no cambiar
    return { winningPattern: scored.find(s => s.p.pattern_id === activePlan.pattern_id)?.p ?? winner, shouldSupersede: false, reason: 'plan_still_valid' }
  }

  return { winningPattern: winner, shouldSupersede: false, reason: 'first_plan' }
}

export async function decideForUser(supabase: SupabaseClient, userId: string): Promise<DecisionOutput> {
  // ... query player_patterns + coach_plans, llamar decide()
}
```

  - `save_plan` invocations: ~1-2 por sesión × 30 sesiones × 2k tokens promedio = ~60k tokens/mes adicionales (sin cache porque cada plan es distinto).

- Usuarios activos mensuales (MAU) > 500
